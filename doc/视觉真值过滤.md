# CARLA 基于 Semantic Segmentation 的有效 Actor 过滤方法（完整流程）

## 一、为什么使用 Semantic Segmentation

在 CARLA 数据采集过程中，我们通常会获取：

- RGB 图像
- Semantic Segmentation 图像
- Depth 图像（可选）
- 当前帧所有 Actor
- 每个 Actor 的 3D Bounding Box

但直接使用 3D Actor 会产生大量无效标注，例如：

- 完全被遮挡的目标
- 图像外目标
- 被建筑/树木遮挡的目标
- 仅极小部分出现在视野中的目标

因此需要进行有效 Actor 过滤。

Semantic Segmentation 提供了一种低成本且高稳定性的可见性判断方法。

---

## 二、整体流程

```
所有 Actor
      │
      ▼
类型过滤（Vehicle / Pedestrian）
      │
      ▼
距离过滤
      │
      ▼
3D Bounding Box 投影
      │
      ▼
得到 Original 2D Bounding Box
      │
      ▼
计算 Truncation（是否超出视野）
      │
      ▼
BBox Clip（裁剪到图像范围）
      │
      ▼
Semantic ROI 统计
      │
      ▼
计算 Visibility Ratio
      │
      ▼
过滤无效 Actor
```

---

## 三、3D Bounding Box 投影

将 Actor 的 3D Bounding Box 投影到图像平面：

得到：

```
xmin, ymin, xmax, ymax
```

此时得到的是：

> Original Bounding Box（未裁剪）

---

## 四、关键步骤：Bounding Box Clip（必须）

### 4.1 为什么必须 Clip

Bounding Box 可能出现：

```
xmin < 0
ymin < 0
xmax > W
ymax > H
```

原因：

- Actor 部分在视野外
- Camera FOV 截断
- 深度投影导致越界

---

### 4.2 Clip 方法

```python
xmin_clip = max(0, xmin)
ymin_clip = max(0, ymin)

xmax_clip = min(W - 1, xmax)
ymax_clip = min(H - 1, ymax)
```

---

### 4.3 无效 Box 过滤

```python
if xmin_clip >= xmax_clip or ymin_clip >= ymax_clip:
    continue
```

说明：

该 Actor 完全不在图像内。

---

## 五、Truncation（截断率）

Truncation 描述：

> Actor 有多少部分在图像外

### 计算方式：

```
Truncation = 1 - (Clip Area / Original Area)
```

其中：

```
Original Area = (xmax - xmin) * (ymax - ymin)

Clip Area = (xmax_clip - xmin_clip) * (ymax_clip - ymin_clip)
```

---

### 示例：

```
Original: 400 × 300
Clip:     200 × 300
```

```
Truncation = 1 - (60000 / 120000) = 0.5
```

---

## 六、Semantic ROI 统计（核心步骤）

### 6.1 提取 ROI

```python
roi = semantic[ymin_clip:ymax_clip, xmin_clip:xmax_clip]
```

---

### 6.2 统计目标类别像素

例如 Vehicle：

```python
vehicle_pixels = count(roi == VEHICLE_LABEL)
total_pixels = roi.size
```

---

## 七、Visibility Ratio（可见率）

### 7.1 定义

```
Visibility = Vehicle Pixels / ROI Area
```

---

### 7.2 示例

```
Vehicle Pixels = 36000
ROI Area       = 40000
```

```
Visibility = 0.9
```

---

### 7.3 推荐阈值

| 类别 | 阈值 |
|------|------|
| Vehicle | > 0.3 |
| Pedestrian | > 0.15 |

---

## 八、Truncation vs Visibility（关键区别）

| 指标 | 含义 |
|------|------|
| Truncation | 是否跑出图像 |
| Visibility | 是否被遮挡 |

---

### ❗ 重要原则

```
Truncation ≠ Occlusion
```

必须分开计算：

- Truncation → 视野外
- Visibility → 视野内遮挡情况

---

## 九、Semantic 方法的本质

Semantic Segmentation 实际只做一件事：

> 在图像可见区域内统计目标类别像素占比

它**不负责处理图像外区域**。

---

## 十、完整算法流程（推荐实现）

```python
for actor in actors:

    # 1. 类型过滤
    if not is_valid_type(actor):
        continue

    # 2. 距离过滤
    if distance(actor) > threshold:
        continue

    # 3. 投影 3D Box
    xmin, ymin, xmax, ymax = project(actor)

    # 4. 计算 Truncation
    original_area = (xmax - xmin) * (ymax - ymin)

    # 5. Clip
    xmin_c = max(0, xmin)
    ymin_c = max(0, ymin)
    xmax_c = min(W-1, xmax)
    ymax_c = min(H-1, ymax)

    if xmin_c >= xmax_c or ymin_c >= ymax_c:
        continue

    clip_area = (xmax_c - xmin_c) * (ymax_c - ymin_c)

    truncation = 1 - clip_area / original_area

    # 6. Semantic ROI
    roi = semantic[ymin_c:ymax_c, xmin_c:xmax_c]

    # 7. 统计目标类别像素
    visible_pixels = count(roi == actor_class_label)

    # 8. Visibility
    visibility = visible_pixels / clip_area

    # 9. 过滤
    if visibility < threshold:
        continue

    # 10. 保存 annotation
    save(actor, truncation, visibility)
```

---

## 十一、Semantic 方法的优点

- 实现简单
- 无需 Depth
- 可快速判断遮挡
- 计算稳定
- 适合大规模数据生成

---

## 十二、Semantic 方法的缺点

### ❗ 关键限制

CARLA Semantic Segmentation：

> 不包含 Instance ID

因此无法区分：

- 同类车辆之间遮挡
- Vehicle A vs Vehicle B

会导致：

```
Visibility 偏高估计
```

---

## 十三、工业级改进方案（推荐）

为了提升标注质量，建议组合使用：

```
Semantic + Depth + 3D Box
```

### 推荐融合逻辑：

- Semantic → 类别判断
- Depth → 前后遮挡
- 3D Box → 几何约束

最终：

```
True Visibility
```

---

## 十四、总结

### 必须遵守的核心原则：

1. Bounding Box 必须 Clip 到图像范围
2. 图像外区域不能参与 Semantic 统计
3. Truncation 与 Visibility 必须分离
4. Semantic ROI 只在 Clip 后计算
5. Visibility 只反映“图像内可见程度”
