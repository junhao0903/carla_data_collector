# 同类别遮挡修正（V1.1关键补丁）

## 1 问题来源

Semantic Segmentation 无法区分同类别实例，例如：

```
Car A（前车）
Car B（后车，被完全遮挡）
```

在 semantic 图像中：

```
ROI 内全部像素均为 Vehicle
```

导致：

```
Car B 被错误认为“可见”
```

---

## 2 核心思想（最小改动原则）

不引入 depth、不引入 ray casting，仅利用：

- ROI 内空间结构
- bbox几何关系
- 连通性分析

---

## 3 Step 1：ROI 内连通域分析（新增）

在 semantic ROI 内构建二值 mask：

```python
mask = (roi == VEHICLE_LABEL)
```

对 mask 做连通域分解：

```python
num_components, labels = connectedComponents(mask)
```

---

## 4 Step 2：判断是否存在“多实例结构”

| 情况 | 含义 |
|------|------|
| num_components = 1 | 可能为单车或完全遮挡 |
| num_components > 1 | 存在多个独立可见实例 |

---

## 5 Step 3：结合 bbox 进行同类遮挡判定

对所有同类 actor（Vehicle）：

### 计算 ROI overlap：

```python
IoU_2D(BBox_i, BBox_j)
```

### 计算深度排序（仅用3D bbox中心z）：

```python
z_i < z_j → i 在前
```

---

## 6 Step 4：遮挡判定规则（关键）

对于目标 B：

### 情况1：完全被覆盖（关键）

满足：

```
IoU(B, A) > 0.8
AND
z_B > z_A
AND
num_components == 1
```

则：

```
B = 被前车完全遮挡 → 过滤
```

---

### 情况2：部分可见

满足：

```
IoU(B, A) > 0.5
AND
num_components > 1
```

则：

```
B = 部分遮挡 → 可保留（低权重）
```

---

### 情况3：独立目标

```
IoU(B, A) < 0.3
```

则：

```
B = 正常目标
```

---

## 7 Step 5：最终过滤逻辑

```text
ROI Semantic
    ↓
Connected Components
    ↓
BBox IoU Graph
    ↓
3D depth ordering (z)
    ↓
Occlusion decision
```

---

## 8 本补丁的作用范围

本方法主要解决：

- 同类车辆遮挡（Car-Car）
- 同类行人遮挡（Pedestrian-Pedestrian）
- 排队场景误检
- ROI semantic 误保留

---

## 9 本方法的本质

这是一个：

> **Geometry-constrained semantic disambiguation**

即：

```
semantic（类别）
+ ROI结构（形状）
+ bbox几何（关系）
```

替代 instance segmentation。

---

## 10 局限性

无法解决：

- 完全像素级重叠（极端贴合）
- 复杂非刚体遮挡
- semantic噪声严重情况

但在 CARLA 中可覆盖：

> 80%+ 同类遮挡误检问题
